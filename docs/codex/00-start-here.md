# Start Here

This repository is meant to teach one practical skill:

> Use Codex to improve a GitHub repository without letting automation become reckless.

## The mental model

Codex is not magic. It is a coding agent that can read files, edit files, run commands, and follow goals. That means it is powerful enough to help and powerful enough to make a mess. Git, CI, branches, and PRs are the seatbelts.

## Your recommended path

1. Learn Git basics.
2. Use `AGENTS.md` so Codex knows the rules.
3. Create a branch before any Codex task.
4. Use `/goal` for substantial work.
5. Run checks.
6. Open a PR.
7. Review before merge.

## Avoid at first

- Auto-merging everything.
- Installing random frameworks.
- Docker-heavy stacks.
- Large local AI models.
- Giving agents full disk access.
- Editing outside the repository.
