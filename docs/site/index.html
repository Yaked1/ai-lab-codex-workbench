Branch, prompt, check, review

AI agent coding without skipping the safety rails.

This offline quick-start site turns the repository's Markdown guidance into a compact path for student-friendly AI coding work: define the task, make a branch, prompt an agent, run checks, review the diff, and merge only when the result is public-safe.

First Safe Loop

Run this loop before letting any agent make multi-file changes. It keeps the work reviewable and gives you a clean rollback point if the agent misunderstands the task.

1

Inspect

Read `AGENTS.md`, check `git status`, and identify the exact files in scope.

2

Branch

Create one focused branch for the task and avoid mixing unrelated edits.

3

Prompt

Give the agent purpose, boundaries, verification steps, and a final report format.

4

Check

Run local checks and fix only the smallest related failure.

5

Review

Review the diff, open a PR, wait for CI, and record user-visible changes.

Local Commands

Use PowerShell from the repository root. These commands are safe validation commands for this repo.

Baseline

Check the current tree

Run this before prompting an agent so you know which changes are yours.

git status
git branch
Validation

Run the repo checks

These commands use Python standard library scripts already present in the repository.

python scripts/repo_health_check.py
python scripts/safe_autofix.py --check
python -m unittest discover -s tests
Autopilot

Inspect generated research status

Use this before triggering scout or local curation work.

.\scripts\local_autopilot.ps1 -Mode status
python scripts/repo_autopilot_status.py

Public Repository Safety

Before you share

  • No `.env`, keys, tokens, passwords, cookies, or private profiles are tracked.
  • No private links, private account IDs, personal emails, or private file paths are present.
  • External tool claims are conservative and marked for official-doc verification.
  • AI-generated diffs have been reviewed by a human.

For fast-changing tools

Verify product behavior, plan access, model availability, pricing, installation commands, platform support, and MCP permissions in official documentation before publishing exact claims.
