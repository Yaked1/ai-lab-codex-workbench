Prompt libraries and setup commands

Build local skills and prompt guides before connecting bigger tools.

Skills, prompt packs, and MCP servers can make agents more useful, but they also widen the trust boundary. Start with local files, placeholders, and read-only workflows.

Safe Setup Principles

Local first

Use repo folders

Keep starter prompt guides in the repository or another public-safe folder before using global config.

Verify

Check official docs

Exact install commands and slash-command behavior change. Verify before running or teaching them.

No secrets

Use placeholders

Never paste API keys, tokens, cookies, passwords, or private URLs into prompts or commands.

SKILL.md-Style Local Skill

Minimal structure

prompt-guides/
  skills/
    docs-review/
      SKILL.md
      examples/
      templates/

What SKILL.md should contain

  • Purpose and when to use the skill.
  • Files or folders the agent should read first.
  • Safety boundaries and commands to avoid.
  • Verification steps and final report format.

MCP Safety Notes

Start here

  • Use read-only servers first.
  • Use test repositories and public data.
  • Document which tools and resources the agent can access.
  • Disable servers when the task is complete.

Avoid early

Avoid write-capable MCP servers connected to private accounts, browser profiles, cloud drives, payment systems, or production databases until permissions and rollback are fully understood.
