End-to-end agent lifecycle

Turn a vague request into a reviewable pull request.

The workflow below is intentionally conservative. It treats agent output as a draft that must pass repository rules, local checks, CI, and human review before merging.

Branch To Merge Workflow

1. Intake

Write the task boundary

State the objective, files likely in scope, files out of scope, success criteria, and checks to run.

2. Branch

Isolate the work

Use one branch per task. Avoid changing workflow YAML, dependencies, or unrelated docs unless requested.

3. Agent

Inspect before editing

Ask the agent to read the relevant files first, then make the smallest correct change.

4. Checks

Run local validation

Run the strongest realistic check for the touched area and fix related failures only.

5. PR

Make the diff reviewable

Summarize what changed, why, commands run, checks run, and limitations.

6. Review

Review like a maintainer

Look for regressions, private data, stale claims, overbroad edits, and missing tests.

PowerShell Command Path

These are safe repository commands. Run them from the repository root. Do not run destructive cleanup, force-push, or dependency installation commands unless the task explicitly requires them.

Start a focused branch

git status
git switch -c agent/short-task-name

Run local checks before a PR

python scripts/repo_health_check.py
python scripts/safe_autofix.py --check
python -m unittest discover -s tests

Review the result

git diff --stat
git diff

Codex Goal Workflow

Goal prompt shape

Use a goal when the task needs persistent execution, validation, and a final report.

/goal Resolve the issue described below.

Objective:
- Make the smallest correct change.
- Keep work inside this repository.
- Run the listed checks.

Scope:
- Include: PATHS_OR_FEATURES
- Exclude: SECRETS, DEPENDENCIES, WORKFLOW_YAML

Verification:
- python scripts/repo_health_check.py
- python scripts/safe_autofix.py --check
- python -m unittest discover -s tests

Final report:
- Summary
- Files changed
- Commands run
- Checks run
- Risks

When to use it

  • The task spans more than one command or file.
  • You need the agent to continue through verification.
  • You want a durable final report for review.
  • You can provide clear safety boundaries.

Claude Code Goal-Style Workflow

Verify current Claude Code slash-command and skill behavior in official documentation before teaching exact syntax. If your setup does not support a `/goal` command, paste the same structure as a normal task prompt. 

# Goal-style task prompt for Claude Code
Read AGENTS.md and inspect the relevant files before editing.

Objective:
Implement REQUEST in the smallest reviewable diff.

Safety:
- Do not read private folders.
- Do not add dependencies.
- Do not modify workflow YAML unless asked.
- Mark fast-changing tool behavior as "verify in official documentation."

Verification:
Run the local checks that apply and report anything not verified.

Agent Task Templates

Tool style Template focus Safety note
Cursor-style IDE agent Ask for a plan, inspect files, then edit visible diffs in the IDE. Review every changed file before accepting.
GitHub Copilot-style agent Start from a small issue, request a PR, and compare CI output with local checks. Do not auto-merge agent PRs.
Antigravity-style agent Use artifact-driven planning and insist on a compact implementation report. Verify current product behavior in official docs.

Merge Gate

Required before merge

  • Diff is focused and understandable.
  • Local checks pass or failures are documented.
  • CI passes before merging to the protected branch.
  • Changelog records user-visible changes.

Stop and review

Stop if the diff includes secrets, private links, generated binaries, lock files, broad deletions, workflow YAML changes, or unverified claims about pricing, plans, models, or platform support.
