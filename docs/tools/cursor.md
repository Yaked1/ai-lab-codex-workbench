# Cursor

Cursor is an AI-focused code editor with agent workflows, codebase chat, planning, rules, skills, MCP support, and CLI-related workflows. It is a good fit for learners who prefer an IDE-style experience.

## Best For

- Planning small issues before edits.
- Explaining files directly in the editor.
- Applying visible multi-file edits.
- Pairing rules or project instructions with agent tasks.
- Trying MCP in an editor environment.

## Beginner Fit

High for users already familiar with VS Code-style editors. The main beginner habit is to review proposed diffs instead of accepting every change.

## Windows Fit

Good for Windows desktop use. Verify the current installer, account, and CLI details from the official docs.

## Surface

Cursor is primarily IDE-based, with CLI and hybrid workflows available depending on setup.

## Main Risks

- Accepting a large generated diff without reading it.
- Giving vague tasks that touch too much of the repo.
- Mixing chat context, rules, and MCP context without knowing which source shaped the output.

## Best First Task

Ask Cursor Agent or Plan mode to create a step-by-step plan for updating one documentation file. Review the plan before allowing edits.
