# Aider

Aider is an AI pair-programming tool for the terminal. It works well when you explicitly choose the files that should be in scope and keep the task small.

## Best For

- Terminal pair programming.
- Focused edits to selected files.
- Git-based local workflows.
- Small features, tests, and docs changes.

## Beginner Fit

Medium. The core workflow is simple, but the user should be comfortable with Git, terminal commands, and file selection.

## Windows Fit

Good when Python and Git are installed. Verify current installation guidance and provider setup before teaching a workshop.

## Surface

CLI, local/hybrid depending on model provider.

## Main Risks

- Adding too many files to context.
- Letting automatic commits hide review steps.
- Provider credential mistakes.
- Running from the wrong directory.

## Best First Task

Run Aider with one Markdown file selected and ask for a small wording improvement. Review the diff before committing.
